Documentation for the Code¶

class seqlib.SeqLib(config)¶

Abstract class for handling count data from a single sequencing library. Creating a SeqLib requires a valid config object, usually from a .json configuration file.

count()¶

This method defines how the data is counted. It must be implemented by each subclass.

enable_logging(log)¶

Messages will be sent to the open file handle log.

load_counts(keys=None)¶

Load the counts from the .tsv files in the self.counts_file dictionary.

The optional keys parameter is a list of types of counts to be loaded. By default, all counts are loaded.

save_counts(directory, keys=None, clear=False)¶

Save the counts DataFrame as a tab-separated file in directory. The counts are saved as type.tsv for each type of counts ('barcode', 'variant', etc.). The file name for each .tsv file is stored in the self.counts_file dictionary, for use by load_counts.

The optional keys parameter is a list of types of counts to be saved. By default, all counts are saved.

If clear is True, saved counts will be set to None after writing. This is used to save memory. If needed later, the counts can be read from the file using load_counts.

set_filters(config_filters, class_default_filters)¶

Sets the filtering options using the values from the config_filters dictionary and class_default_filters dictionary. This method is used by the __init__ method of SeqLib subclasses.

Note

To help prevent user error, config_filters must be a subset of class_default_filters.
